You can visualize the full chromatograms via either the 3D plot, Total Ion Current (TIC), eXtracted Ion Chromatograms (XIC), or Base Ion Chromatogram (BIC). 

# Full Chromatogram:
## 3D Plot: 

The function `chrom_MS1_3D` generates a heatmap of m/z values vs retention times with intensity as color from chromatographic data. The function either samples a set number of points or uses a user defined intensity threshold. The intensities are expressed as log10 to increase the visibility of the low intensity peaks. 

### Inputs

| Argument | Type | Default value | Description |
| :--- | :--- | :--- | :--- |
| `chrom_data` | `Dict` | Required | A dictionary containing the full chromatographic data. The chromatographic data structure, typically generated by MS_Import.jl, which contains the MS1 data ("MS1" key) |
| `n` | `Int-Float64` | 5000 - Optional | The number of points to be plotted or for n less than 1 it is the percent signal to be plotted. |
| `c_lim` | `Float` | `nothing` - Optional | The value for capping the color scale in log10 scale. |
| `interactive`| `Bool` | `false` - Optional | Enables an interactive GLMakie window. |


### Output

* f: A figure object containing the 3D plot of the chromatogram.

### Usage

```julia

# Default settings

chrom_MS1_3D(chrom_data)

# With n set to 50000 
n = 50000

chrom_MS1_3D(chrom_data; n = n)

# With n set as a percentage of the maximum intensity 

n = 0.01 # plotting up to 1%

chrom_MS1_3D(chrom_data; n = n)

``` 

![3D Chromatogram of the demo file](https://gitlab.com/EMCMS_UVA/MZ_Visualization.jl/-/raw/main/figures/chrom_MS1_3D.png?ref_type=heads)

> **Warning**
> Setting the parameter `n` to a very high value (e.g., > 100,000) may significantly increase rendering time and memory usage.
{: .warning }

## TIC: 

The function `get_tic` calculates the Total Ion Chromatogram (*TIC*) from chromatographic data. This function computes the TIC by summing the intensities of all detected ions at each retention time point in the provided chromatographic data. This function does not use the TIC parameter shipped with the XML files as that is for full scan data. In case you are importing a specific mass range for your work this function should give you the *TIC*. 


### Inputs

| Argument | Type | Default value | Description |
| :--- | :--- | :--- | :--- |
| `chrom_data` | `Dict` | Required | A dictionary containing the full chromatographic data. The chromatographic data structure, typically generated by MS_Import.jl, which contains the MS1 data ("MS1" key) |
| `window` | `tuple` of `float64` | `nothing` - Optional | An optional tuple specifying the retention time range (min_rt, max_rt) to plot. |
| `interactive`| `Bool` | `false` - Optional | Enables an interactive GLMakie window. |


### Output

* f: A figure object containing the TIC plot.

### Usage

```julia

# Default settings

get_tic(chrom_data)

# With a set window 
window = (4, 6)

get_tic(chrom_data; window = window)

# With interactivity set to true

get_tic(chrom_data; interactive = true)

``` 

![TIC of the demo file](https://gitlab.com/EMCMS_UVA/MZ_Visualization.jl/-/raw/main/figures/TIC_plot.png?ref_type=heads)

## Multiple TICs: 

The function `get_multiple_tic` calculates the Total Ion Chromatogram (*TIC*) for multiple chromatographic data. 


### Inputs

| Argument | Type | Default value | Description |
| :--- | :--- | :--- | :--- |
| `chrom_data_list` | [`Dict1`,`Dict2`] | Required | An array of dictionaries containing the full chromatographic data. The chromatographic data structure, typically generated by MS_Import.jl, which contains the MS1 data ("MS1" key) |
| `label`| [`String1`,`String2`]  | Required | A list of labels corresponding to each dataset for legend identification. |
| `method` | `overlay` or `stacked` | `overlay` - Optional | A string indicating the plotting method. Options are "overlay" for overlapping plots or "stacked" for vertically stacked plots. |
| `window` | `tuple` of `float64` | `nothing` - Optional | An optional tuple specifying the retention time range (min_rt, max_rt) to plot. |
| `interactive`| `Bool` | `false` - Optional | Enables an interactive GLMakie window. |


### Output

* f: A figure object containing multiple TIC plots.

### Usage

```julia

# Default settings
chrom_data_list = [chrom_1PPB, chrom_100PPB]; # Demo files
labels = ["5 PPB", "100 PPB"];                # Demo labels

get_multiple_tic(chrom_data_list, labels)

# Method stacked

get_multiple_tic(chrom_data_list, labels; method = "stacked")

# With a set window 
window = (4, 6)

get_multiple_tic(chrom_data_list, labels; window = window)


```

# Extracted Chromatograms: 
## MS1 XIC 

The function `get_XIC_MS1` calculates the Extracted Ion Chromatogram (*XIC*) from chromatographic data at the MS1 level. This function computes the XIC by summing the intensities of all detected ions set around a target m/z value with a defined mass tolerance at each retention time point in the provided chromatographic data.  


### Inputs

| Argument | Type | Default value | Description |
| :--- | :--- | :--- | :--- |
| `chrom_data` | `Dict` | Required | A dictionary containing the full chromatographic data. The chromatographic data structure, typically generated by MS_Import.jl, which contains the MS1 data ("MS1" key) |
| `mz_target`| `float` | Required | The target `m/z` value to extract the ion chromatogram for. |
| `mz_tol`| `float64` | 0.005 - Optional | The tolerance range around the target m/z value. |
| `window` | `tuple` of `float64` | `nothing` - Optional | An optional tuple specifying the retention time range (min_rt, max_rt) to plot. |
| `RT`| `Bool` | `true` - Optional | Enables plotting against RT vs scan number. |
| `n_p`| `Int` | 3 - Optional | Number of points to be annotated in the XIC. |
| `offset`| `float` | 0.1 - Optional | An optional offset value to adjust the selection criteria for labeling. |
| `interactive`| `Bool` | `false` - Optional | Enables an interactive GLMakie window. |


### Output

* f: A figure object containing the XIC plot.

### Usage

```julia

# Default settings
mz_target = 195.087

get_XIC_MS1(chrom_data, mz_target)

# With a set window and RT false 
window = (600, 800)

get_XIC_MS1(chrom_data, mz_target; RT = false, window = window, interactive = true)


```

## MS2 XIC


The function `get_XIC_MS2` plots the **XIC** for a target m/z from MS2 data. This function is designed to handle Data Independent Acquisition (DIA) or All Ion Fragment type of experiments where an XIC of an MS2 fragment can be generated.

### Inputs

| Argument | Type | Default value | Description |
| :--- | :--- | :--- | :--- |
| `chrom_data` | `Dict` | Required | A dictionary containing the full chromatographic data ("MS2" key). |
| `mz_target` | `float64` | Required | The target m/z value to extract the ion chromatogram for. |
| `mz_tol` | `float64` | 0.005 - Optional | The m/z tolerance window around the target value. |
| `window` | Tuple{`float64`, `float64`} | `nothing` - Optional | Retention time range (min_rt, max_rt) to zoom the plot. |
| `RT` | `Bool` | `true` - Optional | If true, the x-axis shows retention time; if false, scan number. |
| `n_p` | `Int` | 3 - Optional | The number of top peaks to annotate. Set to 0 to disable. |
| `offset` | `float64` | 0.1 - Optional | Spacing parameter for the sig_select peak labeling utility. |
| `interactive` | `Bool` | false - Optional | Enables an interactive GLMakie window. |

### Output
* `f`: Figure object containing the XIC plot.

### Usage

```julia
mz_target = 304.154

# Default settings

get_XIC_MS2(chrom_data, mz_target)

# With retention time window

get_XIC_MS2(chrom_data, mz_target; window = (4, 10))

# Plot against scan number

get_XIC_MS2(chrom_data, mz_target; RT = false)


```

![XIC MS2](https://gitlab.com/EMCMS_UVA/MZ_Visualization.jl/-/raw/main/figures/XIC_MS2.png?ref_type=heads)
